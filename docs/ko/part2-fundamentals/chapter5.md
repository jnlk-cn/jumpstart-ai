# 제5장 LLM 호출과 프롬프트 엔지니어링

##本章에서 배울 것들

축하합니다! 드디어 진짜 것을 시작합니다!

앞의 몇 챕터에서 많은 "가상적인" 것들에 대해 이야기했습니다 - 업계 인식, 입문 로드맵, 엔지니어링 기초. 이 챕터부터 AI 앱 개발의 **핵심 기술 영역**에 진입합니다.

이 챕터는 이 책의 가장 핵심적인 기술 챕터 중 하나입니다. 원본 내용에서는 기본 기술을 다루었지만, 솔직히 말하면 좀 얕았습니다 - "젓가락 사용 방법"만 가르치지만 "转动丸子怎么夹"는 가르치지 않는 것과 같습니다. 실전에서 정말 유용한 것들을 보충해 드리겠습니다.

이 챕터를 읽고 나면 가져갈 것들:

- **반석 같은 API 호출**: 작동하면 그만인 것이 아니라, 높은 동시성을 처리하고, 스트리밍 출력을 처리하며, 모델이 실패할 때 우아하게降级
- **체계적인 프롬프트 방법론**: 트릭을 외우는 것이 아니라, 왜 이렇게 작성하면 효과적인지, 어떤 기술을いつ使用해야 하는지 이해
- **프로덕션급 구조화된 출력**: JSON Schema怎么做, 다중 레벨 중첩怎么处理, 정보 추출 파이프라인怎么搭建
- **지능형 컨텍스트 관리**: 대화 상태 머신, 의도 인식, 슬롯 채우기 - AI가 진정으로 "이해"하게 함
- **실제 비용 관리**: 이론이 아니라, 한 달에 얼마를 탔는지, 어떻게 절반으로 줄였는지에 대한 경험
- **함정 밟은 피날레**:踏んだ坑, 다시는踏ませたくない
