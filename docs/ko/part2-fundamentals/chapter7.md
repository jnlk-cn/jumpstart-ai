# 제7장 AI 에이전트 핵심 로직

##本章에서 배울 것들

기초편의 마지막 챕터에 오신 것을 축하합니다!

앞의 챕터들이 "AI使用方法"를 가르쳤다면, 이 챕터는 "AI가 혼자 일하게 하는 방법"을 가르칩니다.

먼저 질문합니다: 대규모 모델과 AI 에이전트의 핵심 차이는 무엇입니까?

대규모 모델은 受动的 - 물으면 답하고, 한 질문에 한 답, 천장이 낮습니다. AI 에이전트는 主动的 - 목표를 주면, 스스로 방법을 찾고, 자료를查找하고, 도구를 호출하고, 결과를 검증하며, 작업이 완료될 때까지 계속합니다.

"告诉我怎么做"에서 "你自己看着办"로, 이것은 질적 도약입니다.

이 챕터를 읽고 나면 가져갈 것들:

- **에이전트의 본질 이해**: 왜 에이전트가 AI 앱의 미래인지
- **에이전트 아키텍처 전체 보기**: 인식, 추론, 행동, 기억의 네 가지 주요 구성 요소
- **도구 호출 능력**: AI가 реальный 세계를 조작할 수 있게 함(프로덕션급 코드 포함)
- **계획 및 추론 패턴**: ReAct 완전한 추적, LangGraph 2026 실전, Plan-and-Execute
- **멀티 에이전트 협업**: 두 개의 실제 사용 가능한 사례
- **에이전트 신뢰성 엔지니어링**: 실패의 6가지 패턴, 비용 제어, 출력 검증
