# 제11장 에이전트 개발 실전과 Skill 개발

##本章에서 배울 것들

- 에이전트 비서와 일반 에이전트의 본질적 차이 이해
- 메모리 시스템의 3층 아키텍처 마스터: 단기 기억, 장기 기억, 작업 기억
- 설계 원칙, 인터페이스 사양, 오류 처리를 포함한 완전한 Skill을 0부터 개발하는 방법 학습
- Skill 등록 발견 메커니즘과 조합 워크플로우 설계 방법 이해
- Coze, Dify, LangGraph 3개 프레임워크의 적용 시나리오 비교
- 실행 가능한 AI 개발 비서의 완전한 코드 획득
- 에이전트 비서 평가 시스템과 상용화 경로 이해

2024년은 에이전트 원년입니다. 하지만 다른 이야기를 하고 싶습니다: 2024년은 에이전트 비서 제품 매니저의 악몽의 시작이기도 합니다.

"에이전트"라는 단어가 남용되고 있기 때문입니다. 모든 것이 에이전트라고 불릴 수 있지만, 진정한 에이전트는 아무것도 아닙니다. 그래서 이 챕터에서, 무엇이 진정한 에이전트 비서인지, 일반 에이전트와의 차이가 어디인지, 왜 이 차이가 중요한지를 명확히 하겠습니다.

특히 중점적으로 다룰 것은 Skill 개발입니다 - 에이전트를 "말 잘하는"에서 "일 잘하는"으로 바꾸는 핵심 능력 단위입니다.
