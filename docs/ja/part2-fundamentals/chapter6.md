# 第6章 RAG検索拡張生成

## 本章で学べること

第5章が「怎么让大模型听你的话」を教えたなら、この章は「怎么让大模型知道它原本不知道的事」を教えます。

RAG（Retrieval-Augmented Generation、检索增强生成）はAIアプリ開発で最もコアな技術の一つです。面试的时候80%会问到RAG；做项目的时候十个里有九个需要RAG。理由は简单：模型がどれだけ強くても知识の境界线がある——会社の内部ドキュメント、業務フロー、最新製品データなどを知らない。RAGは这些问题を解決するものです。

しかし、多くの学生はRAGの表面だけを学びました——ベクトル検索知道、文档随便切切、参数随便调调就觉得OK了。面试官深入问一下就会露馅：リランキング怎么做的？このシナリオでなぜセマンティックチャンク而不是递归チャンクを使うのか？Self-RAGとCRAGの違い是什么？マルチモーダルドキュメント怎么处理？

读完这章你会带走：

- RAGの本质を深く理解：不只是「检索+生成」，而是一套解决知识边界问题的系统工程
- テキストチャンクのN种策略をマスター：递归、语义、Parent-Child、そして什么时候用什么
- Embeddingモデルの 선택：2026年中文Embedding哪家强，怎么微调，什么时候需要微调
- 检索戦略を攻略：ハイブリッド検索、リランキング、クエリ書き換え、適応型检索
