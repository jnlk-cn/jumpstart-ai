# 第2章 なぜ「まず乗れ」が合理的な戦略か

## 本章で学べること

前一章读完你已经知道、AIアプリ開発は想像那样するほど高点くない。但是我猜你现在还是有点犹豫——「 барьерは思いのほか低いが、それでも準備ができた的感觉がない」。

 это很正常。我见过太多人在「準備」这件事上一拖就是半年一年，等他觉得准备好了，机会窗口早关了。

この章では明確に説明します：**AIアプリ開発において、「まず乗れ」はギャンブルではなく、最も効率的な合理的戦略です。**

具体的には次ことを理解できます：

- AI技術のスキル半減期がいかに短いか、準備ができた頃には的黄花菜都已凉
- 企業が本当に欲しいのは「全部理解できる人」ではなく「能动く人」
- 边干边学の効率是闭门苦读的2-3倍
- 「私は不够格」という这种感觉是个幻觉，以及如何与之和平共处

---

## 2.1 AIアプリ開発のスキル半減期は驚くほど短い

まず私の亲眼见证した事实を言います。

2022年10月、LangChainが刚发布的时候、私の友人はまるまる3个月把这套框架学透做了十几个demo、履歴書には明確に「LangChainをマスター」と記載。2023年初彼はこれで跳槽に成功し、給与が40%上昇した。

その後呢？2024年、LangChainチームはLangGraphを発表 전혀다른アーキテクチャ。2025年10月、LangChainは直接1.0バージョンを发布 entireフレームワークをゼロから書き直し。さて、どうなるでしょうか？彼が3个月で学んだ内容のほとんどはそのまま無効になった。

**これがAIアプリ開発の残酷な現実：あなたが学んでいるものは、まだ学び终わる前に既に時代遅れになっている可能性がある。**
